# 🔌 Zenith API Documentation

This document outlines the REST API endpoints provided by the Zenith backend server, including authentication flow and task management endpoints.

## 📌 Base URL
By default, the backend server runs locally at:
```http
http://localhost:5000
```

All requests should be sent with the header:
```http
Content-Type: application/json
```

---

## 🔐 Authentication & Session Flow

### 1. Register User
Create a new user account.

- **URL:** `/api/auth/register`
- **Method:** `POST`
- **Body Parameters:**
  | Field | Type | Required | Description |
  | :--- | :--- | :--- | :--- |
  | `fullName` | `string` | Yes | User's display name |
  | `username` | `string` | Yes | Unique login username |
  | `password` | `string` | Yes | Cleartext password (will be bcrypt hashed) |
  | `phoneNumber` | `string` | No | Contact number |

- **Sample Request:**
  ```json
  {
    "fullName": "Jane Doe",
    "username": "janedoe",
    "password": "securepassword123",
    "phoneNumber": "+1234567890"
  }
  ```

- **Responses:**
  - **`201 Created`**: User successfully registered.
    ```json
    {
      "message": "User created successfully"
    }
    ```
  - **`400 Bad Request`**: Username is already in use.
    ```json
    {
      "message": "Username already taken"
    }
    ```
  - **`500 Internal Server Error`**: Database error or validation failure.

---

### 2. Login User (Username/Password)
Authenticate with a username and password to retrieve a JWT session token.

- **URL:** `/api/auth/login`
- **Method:** `POST`
- **Body Parameters:**
  | Field | Type | Required | Description |
  | :--- | :--- | :--- | :--- |
  | `username` | `string` | Yes | The user's username |
  | `password` | `string` | Yes | The user's password |

- **Sample Request:**
  ```json
  {
    "username": "janedoe",
    "password": "securepassword123"
  }
  ```

- **Responses:**
  - **`200 OK`**: Login successful. Returns JWT token and brief user profile.
    ```json
    {
      "token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...",
      "user": {
        "id": "60d5ec49f83f2a3f8c8b4567",
        "username": "janedoe",
        "fullName": "Jane Doe"
      }
    }
    ```
  - **`401 Unauthorized`**: Invalid credentials (username or password).
    ```json
    {
      "message": "Invalid credentials"
    }
    ```
  - **`500 Internal Server Error`**

---

### 3. Google OAuth 2.0 Login
Authenticate using a Google OAuth2 Identity credential. If the user doesn't exist, an account is automatically created using their Google email and display name. If the user already exists with that email, the Google ID is linked.

- **URL:** `/api/auth/google`
- **Method:** `POST`
- **Body Parameters:**
  | Field | Type | Required | Description |
  | :--- | :--- | :--- | :--- |
  | `credential` | `string` | Yes | Google ID Token generated by the Google Sign-In button |

- **Sample Request:**
  ```json
  {
    "credential": "eyJhbGciOiJSUzI1NiIsImtpZCI6IjFhMmIzY..."
  }
  ```

- **Responses:**
  - **`200 OK`**: Authentication successful.
    ```json
    {
      "token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...",
      "user": {
        "id": "60d5ec49f83f2a3f8c8b4568",
        "username": "jane.doe@gmail.com",
        "fullName": "Jane Doe"
      }
    }
    ```
  - **`400 Bad Request`**: No token credential was provided.
    ```json
    {
      "message": "No credential provided"
    }
    ```
  - **`500 Internal Server Error`**: Token validation or profile linking failed.

---

### 4. Check Username Availability
Used dynamically in the registration UI to check if a username is already taken.

- **URL:** `/api/auth/check-username/:username`
- **Method:** `GET`
- **Path Parameters:**
  - `username` (`string`): The username to check availability for.

- **Responses:**
  - **`200 OK`**: Returns availability boolean status.
    ```json
    {
      "available": true
    }
    ```

## 📝 Todo Tasks Endpoints
> [!IMPORTANT]
> All Todo endpoints are protected and require a valid JWT token. You must include the token in the `Authorization` header as:
> `Authorization: Bearer <your_jwt_token>`

### 1. Get All Tasks
Retrieve all task items belonging to the authenticated user. Results are sorted by creation date descending.

- **URL:** `/api/todos`
- **Method:** `GET`
- **Headers:**
  - `Authorization: Bearer <token>`

- **Responses:**
  - **`200 OK`**: Returns list of todos.
    ```json
    [
      {
        "_id": "60d5ec89f83f2a3f8c8b4570",
        "userId": "60d5ec49f83f2a3f8c8b4567",
        "text": "Complete project documentation",
        "completed": false,
        "date": "2026-06-29",
        "createdAt": "2026-06-29T14:00:00.000Z",
        "__v": 0
      }
    ]
    ```
  - **`401 Unauthorized`**: Token missing or invalid.
  - **`403 Forbidden`**: Token expired.

---

### 2. Create Task
Create a new task scheduled for a specific date.

- **URL:** `/api/todos`
- **Method:** `POST`
- **Headers:**
  - `Authorization: Bearer <token>`
- **Body Parameters:**
  | Field | Type | Required | Description |
  | :--- | :--- | :--- | :--- |
  | `text` | `string` | Yes | The description of the task |
  | `date` | `string` | No | Scheduled date in `YYYY-MM-DD` format (defaults to server's today) |

- **Sample Request:**
  ```json
  {
    "text": "Publish Zenith to GitHub",
    "date": "2026-06-30"
  }
  ```

- **Responses:**
  - **`201 Created`**: Task created successfully.
    ```json
    {
      "_id": "60d5ec99f83f2a3f8c8b4571",
      "userId": "60d5ec49f83f2a3f8c8b4567",
      "text": "Publish Zenith to GitHub",
      "completed": false,
      "date": "2026-06-30",
      "createdAt": "2026-06-29T14:15:00.000Z",
      "__v": 0
    }
    ```
  - **`400 Bad Request`**: Missing required parameters.
  - **`401 Unauthorized`**

---

### 3. Update Task
Modify properties of an existing task (e.g. mark complete, edit text description, or reschedule to another day).

- **URL:** `/api/todos/:id`
- **Method:** `PATCH`
- **Headers:**
  - `Authorization: Bearer <token>`
- **Path Parameters:**
  - `id` (`string`): The MongoDB object ID of the task to update.
- **Body Parameters (Provide at least one):**
  | Field | Type | Description |
  | :--- | :--- | :--- |
  | `completed` | `boolean` | Mark completion status |
  | `text` | `string` | Change task text |
  | `date` | `string` | Reschedule date in `YYYY-MM-DD` |

- **Sample Request:**
  ```json
  {
    "completed": true
  }
  ```

- **Responses:**
  - **`200 OK`**: Task updated successfully. Returns updated object.
    ```json
    {
      "_id": "60d5ec89f83f2a3f8c8b4570",
      "userId": "60d5ec49f83f2a3f8c8b4567",
      "text": "Complete project documentation",
      "completed": true,
      "date": "2026-06-29",
      "createdAt": "2026-06-29T14:00:00.000Z",
      "__v": 0
    }
    ```
  - **`400 Bad Request`**: Validation or server error.
  - **`404 Not Found`**: Task doesn't exist or is not owned by the authenticated user.
  - **`401 Unauthorized`**

---

### 4. Delete Task
Permanently remove a task.

- **URL:** `/api/todos/:id`
- **Method:** `DELETE`
- **Headers:**
  - `Authorization: Bearer <token>`
- **Path Parameters:**
  - `id` (`string`): The MongoDB ID of the task to delete.

- **Responses:**
  - **`200 OK`**: Task deleted.
    ```json
    {
      "message": "Todo deleted"
    }
    ```
  - **`404 Not Found`**: Task doesn't exist or is not owned by the authenticated user.
  - **`401 Unauthorized`**
